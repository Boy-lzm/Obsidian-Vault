下面的代码位于bootloader flash driver
```
#pragma section=".funcvect"
#pragma location=".funcvect"
__root const uint32 func_vect[] = 
{
    (uint32)Pflash_EraseSector,
    (uint32)Pflash_Program,
    (uint32)Dflash_EraseSector,
    (uint32)Dflash_Program,
    (uint32)Pflash_WriteUpperVersionValue,
};

#pragma section=".validflag"
#pragma location=".validflag"
__root const uint32 valid_flag = 0x6BFD8912U;
```
该段程序定义了一个函数指针表和一个有效标记

1. #pragma 指令与段定义

		#pragma section:指定后续变量将被放入名为.funcvect的段中。段位链接器将数据或代码组织到指定内存区域的基本单位
		#pragma location：指定该段的具体放置位置（在连接器脚本中定义）。此处表示.funcvect段将会被放到连接器脚本定义的.funcvec内存区域中

2. _ _ root关键字
	IAR编译器的扩展关键字，用于强制链接器保留该变量，即使该变量没有被显示引用。如果没有改关键字链接器可能会优化没有使用的变量。这里明确要求保留函数向量表，因为他将被bootloader或其他模块通过固定地址访问

3. 函数向量表func_vect
	这些函数集中放在一个已知的固定的内存位置，可以让应用程序或bootloadr通过索引（而非直接函数名来调用他们）
	
4. 有效标志位valid_flag
	定义一个固定的魔数0x6BFD8912U，存储在名为.validflag的段中，bootlaoder启动时检查该地址的值是否等于0x6BFD8912U，以判断.funcvect是否有效
