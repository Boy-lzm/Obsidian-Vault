**ELF/OUT/HEX**
	ELF和OUT算可执行文件的格式，HEX算是程序的镜像文件 
	 编译汇编生成 .o ->链接生成elf -> 下载工具调试器（jlink，STlink）识别ELF找到.text .data找到地址，提取纯数据写入flash
		
- ELF(编译链接后标准产物)
	- 包含：
		- 包含代码段（.text)
		- 数据段（.data/.bss）
		- 符号表（函数名，变量名）
		- 调试信息
	- 结构复杂
	- 用于调试（断点，单步）
- HEX(下载用的镜像文本)
	- 文本格式（SACII）
	- 带地址信息
	- 每一行一条记录
- OUT类似ELF


**内存分配**
* .test：函数翻译成二进制指令放在代码段中
* .data: 初始化的全局变量和静态局部变量放在数据段中
* BSS段比较特殊，一般来讲，未初始化的全局变量和静态变量会放置在BSS段中，但是因为它们未初始化，默认值全部是0，其实没有必要再单独开辟空间存储，为了节省存储空间，所以在可执行文件中BSS段是不占用空间的。但是BSS段的大小、起始地址和各个变量的地址信息会分别保存在节头表section headertable和符号表.symtab里，当程序运行时，加载器会根据这些信息在内存中紧挨着数据段的后面为BSS段开辟一片存储空间，为各个变量分配存储单元。
* 函数内部定义的static变量与BSS类似
* .rodata 程序中定义的一些字符串、printf函数打印的字符串常量则放置在只读数据段.rodata中。

* **变量存放位置**
1. 全局const：通常放在.rodata
```
const int a = 10;
```
2. static const：还是放在.rodata, 不过对外部不可见
```
static const int b = 20;
```
3. 局部const：不优化时放在栈上；优化后做为立即数，不占内存
```
void func()
{
    const int c = 30;
}
```
4. static const数组：一定在.rodata
```
static const uint8_t table[] = {1,2,3,4};
```
5. 局部static：编译时初始化，初始化0和非0分别放在.bss和.data

预处理器：将源文件main.c经过预处理变为main.i。
编译器：将预处理后的main.i编译为汇编文件main.s。
汇编器：将汇编文件main.s编译为目标文件main.o。
链接器：将各个目标文件main.o、sub.o链接成可执行文件.out

**编译**
从C源文件到汇编，实际就是将C中的程序代码块，函数转换为汇编中的代码块；将C程序中的全局变量，静态变量常量转化为汇编中的数据段，只读数据段
 1. 词法分析
	 扫描文件，将字符识别位最小记号单元token，
	 常见token：
		- 关键词：int， float，for，while，break
		- 标识符：函数名，变量名，标号
		- 字面量：数字，字符串
		- 运算符：减价乘除等等
		- 分隔符：程序结束分号“；”，for循环中，
	以下代码分解为8个token：token：“sum” “=” “a” “+” “b” “/” “c” “；”
```
		sum = a + b / c;
```
 2. 语法分析
	 对token序列分析，发现不能构建的语句或表达式，比如for循环少了；就报错：syntax error
3. 语义分析
	对语法分析输出结果进行判断，如果传递给函数的实参与函数声明的形参类型不匹配，或者使用一个为定义的变量
