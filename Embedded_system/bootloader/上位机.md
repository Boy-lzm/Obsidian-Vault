1. 分析SREC文件
	1. 读取一行 
		- 长度不应该小于8：因为数据前面的类型之类就是8了
		- 第一位应该是S
		- 获取类型：该位表示这行的数据类型
			- 0表示header：该srec文件的名称
			- 1表示地址长度为16位（0xFFFF）
			- 2表示地址长度为24位（0xFFFFFF）
			- 3表示地址长度为32位（0xFFFFFFFF）
			- 7，8，9分别表示32，24，16位程序起始地址
		- 再获取长度，地址和数据和checksum，并将所有信息做为一个srecord返回
		-*一个srecord中包含（0，type，address, datalen, data, checksum）*
	2. 当类型位1，2，3时处理该行数据
		- 如果ssrecordBySection（总的srecord数组，默认长度为0）长度为0，则在该行后面加上地址偏移0，并将改行加入到总的ssrecordBySection数组中。
			那么第一个srecord为：（0，type，address, datalen, data, checksum，0）*
		- 如果ssrecordBySection长度不为0
			- 获取上一条srecord和当前srecord的地址，计算出地址偏差。
			- 获取上一个srecord的地址偏差（最后一个元素）
			- 如果上一个srecord地址偏差为0
				- 如果当前srecord和上一个srecord地址偏差为16
					 - 则把16放到当前srecord末尾（做为该srecord的地址偏差）
					 - 把当前srecord放入ssrecordBySection
				- 如果当前srecord和上一个srecord地址偏差不为16
					表示当前的srecord的地址和之前的地址不连续了
					- 则把0放到当前srecord末尾（做为该srecord的地址偏差）
					- 把当前srecord放入ssrecordBySection
			- 如果上一个srecord地址偏差为16
				- 如果当前srecord和上一个srecord地址偏差为16
					 - 则把16放到当前srecord末尾（做为该srecord的地址偏差）
					 - 把当前srecord放入ssrecordBySection
				- 如果当前srecord和上一个srecord地址偏差不为16
					- 则把0放到当前srecord末尾（做为该srecord的地址偏差）
					- 把当前srecord放入ssrecordBySection


		数据处理的流程
		1. 按行分析srec文件，输出该行srec文件的：{0, type(1,2,3),  address, datalen, dataarr, checksum}，记作srecord
		2.  将这个srecord放入总的container中
		3. 如果当前srecord和 上一个srecord的偏移地址不是16，那么则将之前容器中所有srecord进行处理和发送
		4. 如果type是1，2，3
			1. 将该行输出信息加入到总信息srecordBySection中
			2.  如果这行的地址与上行地址的差不是16，那么表示srec文件地址不连续，之前的所有srecord算做一个section，对之前所有的srecord进行处理
				1. 根据上一段srecord，进行section end essage打包
					1. 设置ection end essage长度为固定为13
					2. 获取上一段srecord（section）的起始地址，放入section end essage中
					3. 获取上一段srecord（section）所有data（仅数据部分）的长度，放入section end essage中
					4. 计算上一段srecord（section）所有data（仅数据部分）的CRC，放入section end essage中
					5. 加入section end essage的checksum
					6. 返回section end essage
				2. 发送section end message
					1. 单个host messgae的构建，以及组成host msgs
						1. 加入LIN NAD（仅LIN通信）
						2. 加入host message id
						3. 加入index
						4. 如果剩余的section messge的长度大于一次发送所包含的数据长度（LIN：5+NAD； CAN：6； CANFD：62），那么从section messge中拿出指定的长度放到这个host msg中。
						5. 如果最后一个host message长度没有装满，那么补0
						6. 将这个host msg放到host msgs中
					
					2. 依次发送host msg
						1. 发完之后等待回读
						2. 判断回读的bl id 和host id

parseSrecord的作用：
	其入参是srec文件的某一行
	返回改行包含的信息，返回格式为{0，type，address，datalen，data，checksum}
		{0, 2, {01,10,00}, 16, {00,40,00,20,45,39,01,00,C7,29,01,00,FB,2B,01,00}, E3}
processSrecord的作用：
	将每一行的srecord加入到container中
SrecordToStr的作用：
     其入参是srec文件的某一行
     返回该行包含的信息，格式为{0，type, address, datalen, data, 0}
	     {0, 2, {011000}, 16, {10, 01, 14, S2, 00 40 00 00, 01 39 45 20 01 29C700}, E3}
processHostSectVerifs：
	仅用来发送一个section结束后 一个section end message， 晚于之前所有srecord的发送
