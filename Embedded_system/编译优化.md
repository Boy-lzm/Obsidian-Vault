1. 指令重排
	M4是流水线处理方式，
2. 将变量放到寄存器中，减少内存访问。
	 1. 导致读取时无法获取实际内存的值


寄存器缓存
	现象：
	int flag = 0;
	
	while(flag == 0) {
	}
	编译器优化后：
	// flag只读一次，放寄存器
	while(1);

指令重排
	现象：
	data = 100;
	flag = 1;
	编译器可能：
     flag = 1;
	data = 100;
		结果：在中断中
		if(flag)
    use(data);   // data 还没写好
    
死代码消除：
	REG = 1;
	REG = 2;
	编译器可能：
	REG = 2;
	 结果：
	 - 外设初始化失败
	- 时序丢失

循环优化：
	for (i = 0; i < 1000; i++);
	编译器可能直接删除，导致延时消失
	

原因：编译器假设：程序没有未定义行为，且变量不会被外部改变，可通过加volatile消除

volatil作用：1. 每次都从内存中读取，2. REG = 1;  REG = 2;都不会被优化
